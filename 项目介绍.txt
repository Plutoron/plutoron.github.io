我在滴滴主要负责的项目是简历中的 质量效能平台 PC 端的项目
这个平台是我 从0开始搭建的
项目分为 前端 和 node 两部分 
前端 是 使用 webpack 打包的 vue 框架的单页面应用 
node 是 滴滴 内部的一个node 框架
node层 主要用来 对接 统一sso登陆 统一审批流程 以及 平台内部 用户和权限管理
部署打包的话 是 前端打包成bundle文件 通过node服务托管静态资源 部署
平台的业务 主要分为两部分
第一个是 线下数据库的 申请 和 在线查询。
在线查询功能的话我是 使用 codeMirror 实现的 在线编辑器

第二个是 提供持续集成的能力，我们平台对接 内部 gitlab
用户可以把gitlab上的项目关联到质量效能平台上
在平台上 通过 在线编辑 或者 图形化 界面 配置 CI/CD 流程，比如 打包 部署 发布 等阶段的配置

另外，因为我主要的项目经历是 中后台项目。
所以，在业务开发过程中，根据自己的经验呢。
也是沉淀了 vue 场景下 中后台项目的通用业务场景，
比如 表格展示 + 分页 + 搜索这种场景的 代码模板和通用组件
比如 弹窗 + 表单提交的场景，的交互，提交是 按钮 loading 提交成功 及 提交失败的 反馈 等
这样提高了开发效率，也让整个产品更加整体
